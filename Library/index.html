Objective

The objective of this project was to design and create a virtual library database system using a spreadsheet. This system was intended to help track information about books in a digital library, including details like book IDs, titles, authors, genres, availability, and checkout dates. The main goal was to understand the use of databases and how to structure data for effective management and retrieval.

Key Skills Acquired

  • Basic Spreadsheet Work: I learned how to use spreadsheets for organizing and managing data efficiently.
  • Data Structure and Design: I gained knowledge in how to structure data within tables, ensuring each piece of information is appropriately categorized and linked.
  • Database Management in Everyday Systems: I understood the importance of databases in real-world systems, especially for managing large volumes of data in an organized manner.

Steps and Instructions Followed
1. Created the Database Structure:

I created a spreadsheet with the following columns to represent the necessary data about books:

  • book_ID(Primary Key): A unique identifier for each book in the library.
  • title: The name of the book.
  • author: The author of the book.
  • genre: The genre or category of the book.
  • publication_date: The publication year of the book (as a number).
  • available: A field indicating whether the book is available or checked out (boolean values: Yes/No).li>
  • last_checkout_date: The date the book was last checked out (if applicable).

2. Validate Information:

After creating the database structure, I populated the spreadsheet with data for 50 books, including relevant details such as book titles, authors, genres, and availability statuses. I then saved the data as a .CSV file and uploaded it to the virtual library platform.

3. Identified Issues with the Database Structure:

Upon reviewing the initial database structure, I identified a few potential issues:

  • Lack of Detailed Author Information: The author information was limited to the author‚Äôs name only. There was no space to include additional author details like nationality, biography, or birthdate.
  • No Separation of Genres: The genre field was a simple text field, but it could benefit from a separate table for genres, which would allow for better categorization and future scalability.
  • Missing User Information for Checkout History: The ‚Äúlast_checkout_date‚Äù column did not link back to a specific user, which is critical for tracking who checked out a book.

4. Restructure the Database:

To improve the database structure, I made the following adjustments:

  • Added an Author Table: I created a separate table for authors to store additional information such as their birthdate and nationality.
  • Created a Genre Table: I created a new table for genres, allowing each book to reference a genre by its unique ID. This would prevent the duplication of genre data across multiple books and improve the overall organization.
  • Created a Checkout Table: I introduced a checkout table to track which user borrowed a book and when it was checked out. This would allow for better tracking and reporting on library usage.

5. Consideration of Primary Keys:

In addition to the book_ID as the primary key for the book table, I used:

  • author_ID as the primary key for the author table.
  • genre_ID as the primary key for the genre table.
  • checkout_ID as the primary key for the checkout table.
Each table now had a unique identifier, which ensured the integrity of data and relationships between tables.
